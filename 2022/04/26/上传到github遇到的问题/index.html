连接失败的解决方法

1、出现Not a git repository..
fatal: Not a git repository (or any of the parent directories): .git

使用该命令

 git init
2、设置github的token

参考链接

3、出现如下代码
fatal: unable to access 'https://github.com/.......': OpenSSL SSL_read: Connection was reset, errno 10054

使用如下命令

git config --global http.sslVerify "false"
4、出现如下代码
emote: Support for password authentication was removed on August 13, 2021. Please use a personal access token instead.
remote: Please see https://github.blog/2020-12-15-token-authentication-requirements-for-git-operations/ for more information.

解决方案：

首先设置token，参考第2步

git remote remove origin（删掉现在的远程地址）
git remote -v （查看一下是否删除成功，没有输出任何内容就是删除成功）
git remote add origin https://<your_token>@github.com/<USERNAME>/<REPO>.git

注意：

  • <your_token>：换成你自己得到的token
  • <USERNAME>：是你自己github的用户名
  • <REPO>：是你的仓库名称
